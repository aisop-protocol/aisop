# Contributing to AISOP

Thank you for your interest in contributing to the **AI Standard Operating Protocol (AISOP)**! 

As an open standard, we value contributions from the entire AI community. Whether you are fixing a typo in the spec, porting the SDK to a new language, or proposing a major protocol change, you are welcome here.

## How to Contribute

### 1. Reporting Issues
- Use GitHub Issues to report bugs in the Spec, Schema, or SDKs.
- Please verify if the issue is already reported.

### 2. Proposing Changes (RFC Process)
Major changes to the protocol (e.g., adding a new standard action or changing the JSON structure) must go through the **RFC (Request for Comments)** process.

1.  Copy `rfcs/0000-template.md`.
2.  Write your proposal.
3.  Submit a Pull Request with your new RFC file.
4.  The community will discuss and vote.

### 3. Contributing Code (SDKs)
- All code contributions must pass existing tests.
- Please assist in writing tests for new features.
- Follow the coding style of the respective language SDK.

## Code of Conduct

We are committed to making participation in this project a harassment-free experience for everyone. Please be respectful and professional.
